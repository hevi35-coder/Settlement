# 사용자 인증 구현 및 Settings 테이블 연동

**Task ID:** T-001
**Status:** BACKLOG
**Importance:** MUST
**Complexity:** 7/10
**Urgency:** 9/10
**Dependencies:** None

## Description

### 요구사항
- 이메일/비밀번호 로그인 기능 구현
- 구글/애플 소셜 로그인 기능 구현
- 로그인된 사용자와 Settings 테이블 연동하여 월급 및 고정지출액 로드·저장
### 구현 세부사항
- Next.js App Router 내 인증 페이지 및 API Routes(/api/auth/*) 구성
- @supabase/supabase-js Auth 모듈 사용 및 환경변수 NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY 설정
- RLS(Row Level Security) 적용으로 사용자별 데이터 접근 제어 설정
### 테스트 전략
- 단위 테스트: Supabase Auth 클라이언트 모킹 후 로그인/로그아웃 시나리오 검증
- 통합 테스트: API Routes 인증 흐름과 Settings CRUD 요청 검증
- E2E 테스트: Cypress 또는 Playwright로 로그인 후 설정 페이지 접근 및 저장 동작 확인
